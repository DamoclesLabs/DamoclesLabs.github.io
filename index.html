GameFi Security Shouldn't only have contract audit

The security construction of GameFi should absorb the security construction experience of tranditional Game, and expand the content with the business, The wider the breadth, the greater the security

The current security company protection for GameFi is limited to contract audits and code audits, but with our team's many years of experience with anti-cheat, only security audits are inadequate. Because in the game territory, there are a variety of ways to cheat, including the studio, cheating, protocol hang-ups and simulated keys. These cheating means can not be avoided through a simple audit, game anti-cheating is a territory that requires long-term confrontation.

In case of the complex and diverse cheating methods in the game territory, our team provides a variety of countermeasures, including detection schemes for traditional functional cheating, advanced Protection for the game code, for the studio account traceability identification scheme. And we invite all project builders to co-construct the credit sub-scheme, details of which can be found in our white paper.

More real user portrait

How to identify users and user groups is a difficult point in the web3 territory

We will specify the contract's interaction object as the unit of analysis, the behavior sequence is modeled for the interactive users, and the model is analyzed by clustering.
